# Contributing to drugsideeffect

Thank you for your interest in contributing to **drugsideeffect**!

We welcome contributions that improve the code, documentation, usability, and research transparency of this project.

## Ways to Contribute

You can contribute by:

- Reporting bugs
- Suggesting new features
- Improving documentation
- Adding or refining lexicons
- Improving visualizations
- Submitting code improvements
- Writing tests

## Reporting Issues

If you find a bug or want to request a feature:

1. Check existing Issues first.
2. Open a new Issue with:
   - A clear description
   - Steps to reproduce (if applicable)
   - Expected vs. actual behavior
   - Python version and OS

## Submitting Pull Requests

1. Fork the repository
2. Create a new branch:


3. Make your changes
4. Ensure code runs without errors
5. Submit a Pull Request with:
- Clear explanation of changes
- Reference to related Issue (if any)

## Adding to Lexicons

Lexicons are located in the package modules as Python dictionaries/lists.

When contributing:
- Clearly document new entries
- Ensure mappings are medically neutral
- Do not imply clinical validation

## Testing

Before submitting:
- Verify that `sideeffect_pipeline()` runs successfully
- Confirm visualizations generate without errors

## Code of Conduct

Please be respectful and constructive in discussions and reviews.
